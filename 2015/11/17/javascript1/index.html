JavaScript笔记

  • JavaScript
  • 前端

自我剖析反省

我感觉现在学知识有点浮躁，恨不得很快就把一门知识掌握，比如现在:JavaScript还刚看一遍就急急忙忙的去学jQuery、json、bootstrap而且总是半途而废，每天看着总是忙忙碌碌的，但最终感觉自己并没有学多少东西，所以从现在开始我要开始沉下心来学习了，为了解决我每次看书看多少忘多少的问题，我准备用写心得体会的方式来解决这个问题，不知道这个办法有木有用，但是至少能让我浮躁的心稍微安静下来写写心得，相比以前还是有点点作用的。网上查了JavaScript语言的入门书籍最经典的就是《JavaScript Dom 编程艺术》，一次偶然的机会得知我同学有一本于是我就借过来准备仔细研读了。以下是我研读的心得体会（感觉自己废话好多呀 ）

  • 本书分为12章：第一章便是每本书的传奇历史了，这本书也不例外讲了JavaScript的简史，此章我就大概看看，草草的略过。
  • js代码的注释：//是单行注释 也可以用 <！–注释 来表示
    /* 内容*/ 是多行注释   <！-- 注释 -->
    这里留一个一直没有搞懂的问题： js、html、css注释区别
  • js变量问题：首先绝对要区分大小写（印象中没有哪一门语言变量不用区分大小写的） 变量中不允许有空格和标点符号（不懂为啥不能用，但是我也没有那习惯用，我一般用的就是加下划线）
  • js中的字符串单引号双引号都可以用，以前一直在就纠结这个，发现自己每次纠结都用对了，我还以为是自己运气好呢~
  • 转义字符“\” var mood =’don\’t’;
  • js定义数组 var a=Array(); js数组里面变量类型可以混着用
    创建对象 var a=Object();
    a.name="liangbijie";
a.age=19;
    还有一种方式就是 var a={name:”liangbijie”,age=19}
  • 节点：一共三种节点：元素节点、文档节点和属性节点
    元素节点：就是各种标签啥的都是元素节点
    文档节点：P标签里面的内容就是文档节点
    属性节点：p标签里面title=啥啥啥 title那个就是一个属性节点
  • 获取元素的方法：有三种
    第一种：getElementById(id) 有名字就可以猜测到他肯定就是用来获取有id的元素的
    第二种：getElementByTagName(tag) 获取一个对象数组,返回的是一个数组
    第三种：getElementByClassName（class）这个方法比较新，慎用。他返回的也是一个数组
    getElementByClassName（class1 class2）这个获取的就是同时含有class1和class2的函数
    此段学到了P42页的3.4.6
  • 接下来获取和设置属性
    第一种：getAttribute
    此方法只能通过元素节点对象调用 ，如果对象没有属性值就会返回一个null
    我们可以利用这个特性来检查某项数据是否是null；
