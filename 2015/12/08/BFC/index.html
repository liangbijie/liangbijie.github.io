揭秘BFC，简单粗暴（未完）

  • CSS
  • HTML
  • 前端

导读：本文讲了什么是BFC，以及BFC的布局规则

什么是BFC？

在介绍BFC之前我先来介绍下Box(盒子)和Formatting Context（一个独立的渲染区域）

Box(盒子)

Box(盒子)是CSS布局的对象和基本单位，直观点来说，就是一个页面是由很多个Box(盒子)组成的,你看到的页面其实是由各种盒子组装在一起所展示的效果。
元素的类型和display属性，决定了这个Box的类型。不同类型的 Box，会参与不同的 Formatting Context（一个独立的渲染区域），因此Box内的元素会以不同的方式渲染。
接下来我介绍下盒子的类型

  • block-level box:
       这个盒子的display 属性为block/list-item/table，并且参与 block fomatting context；
  • inline-level box:
       这个盒子的display 属性为 inline/inline-block/inline-table，并且参与 inline formatting context；
  • run-in box:
       css3中才有，这儿先不讲了。

    Formatting context（一个独立的渲染区域）

    Formatting context 是 W3C CSS2.1 规范中的一个概念。它是页面中的一块渲染区域，并且有一套渲染规则，它决定了其子元素将如何定位，以及和其他元素的关系和相互作用。
    最常见的 Formatting context 有
  • Block fomatting context (简称BFC)
  • Inline formatting context (简称IFC)
  • CSS2.1 中只有 BFC 和 IFC, CSS3 中还增加了 GFC 和 FFC

BFC是什么鬼？

BFC的全称是Block formatting contexts，网上都翻译成块级格式上下文而我觉得翻译成块级格式化范围更为恰当
它是一个独立的渲染区域，只有Block-level box参与， 它规定了内部的Block-level Box如何布局，并且与这个区域外部毫不相干。
通俗的来解释BFC，他是一个独立的布局环境，我们可以理解为一个箱子（实际上是看不见摸不着的），箱子里面物品的摆放是不受外界的影响的。
转换为BFC的理解则是：BFC中的元素的布局是不受外界的影响（我们往往利用这个特性来消除浮动元素对其非浮动的兄弟元素和其子元素带来的影响。）并且在一个BFC中，块盒(block-level box)与行盒(inline-level box)都会垂直的沿着其父元素的边框排列。

BFC布局规则

  • 规则一：内部的Box会在垂直方向，一个接一个地放置。
  • 规则二：Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠
  • 规则三：每个box的左边，都会与包含块box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
  • 规则四：BFC的区域不会与float box重叠。
  • 规则五：BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  • 规则六：计算BFC的高度时，浮动元素也参与计算

哪些元素会生成BFC?

  • 根元素
  • float属性不为none
  • position为absolute或fixed
  • display为inline-block, table-cell, table-caption, flex, inline-flex
  • overflow不为visible

BFC的作用及原理

自适应两栏布局

代码：

<style>
    body {
        width: 300px;
        position: relative;
    }

    .aside {
        width: 100px;
        height: 150px;
        float: left;
        background: #f66;
    }

    .main {
        height: 200px;
        background: #fcc;
    }
</style>
<body>
    <div class="aside"></div>
    <div class="main"></div>
</body>
