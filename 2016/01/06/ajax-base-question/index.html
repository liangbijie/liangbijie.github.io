AJAX之一些基本问题的解决办法

  • AJAX
  • 前端

导读：AJAX我们或多或少会碰到一些问题，这里我总结了一些常见的ajax问题和其解决方案

IE下的缓存问题

问题描述：
如果我们要通过ajax用GET方法去多次重复的申请同一个url的数据，在IE下我们第一次申请的数据就会缓存，第二次申请的时候服务器状态码就会是304，直接读取缓存中的数据，此时就是不能读取到服务器最新的数据。
解决方案：
分析：
一：使每次GET的url都不同，这样就不会去读取缓存中的文件了。此时我们就可以利用每次发送的时间这个实时变化的变量来解决这个问题了，直接在url后面加一个时间戳或者一个随机数，使url变成独一无二的，这样IE下就不会出现缓存问题了。
二：将GET方法变成POST方法，就不会出现这个问题了。
操作：
xhr.open("get","xxxx.aspx?_dc="+new Date().getTime(),true);

ajax对象属性的大小写问题

问题描述：
在w3c浏览器，如ff中，对大小写是敏感。如 if(xhr.readystate==4)这种写法，在ie下是成立的，但是在火狐下就行不通了，因为ie不区分大小写，ff是区分大小的。
解决方案：
将代码改成标准的写法：if(xhr.readyState==4)，同理还有属性 responseText，responseXML。

ajax本地调试问题

问题描述：
有些时候我们需要再本地测试ajax代码，一直不执行回调函数（即xhr.status==200的代码）
解决方案：
分析：
xhr.status==200是要通过服务器页面没有发生错误或者转向时才返回200状态的，在本地测试是没有连接服务器的。但是我们可以在本地测试的时候，如果页面未发生错误，此时的xhr.status是0，不是200。所以可以多加一个xhr.status==0的判断。
操作：

if(xhr.status==200||xhr.status==0){
  alert('ok');
}
