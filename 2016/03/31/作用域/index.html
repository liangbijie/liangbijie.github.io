作用域<一>初始作用域

  • 作用域
  • 前端

导读：通过一些浅显易懂的例子初步了解什么是作用域

一个小例子

①：

alert(a);   
var a = 1;

上面这个段js代码会弹出来一个undefined
②：

alert(a);   
a = 1;

上面这个段js代码会报错，为什么呢？这就是由于js作用域影响产生的结果，下面我们就来看看作用域到底是个什么东西~

js解析器

在我们了解作用域前，我们先了解浏览器解析js代码的机制（这部分功能是由浏览器 js解析器 负责的）浏览器有一个专门负责解析js语句的东西，我们姑且叫做“js解析器”

解析机制：

第一步：预解析（提前找一些东西，先初始化一个值）
找什么东西呢：
1、var；2、function；参数
附什么值呢：
1、当浏览器看到var 就给其参数赋值为undefined
2、当浏览器看到function 就将此函数的代码块作为字符串赋值给这个函数
3、当浏览器看到参数
第二步：逐行解读代码
从哪开始解析：当预解析完成后，文件从第一行代码开始解析代码
怎么解析：逐行运行每行代码，当遇到函数调用的时候，则进入函数，在函数内部又重新运用这套解析机制

实例一

alert(a)
alert(fn1)
var a = 1;
fn1();
function fn1(){
    alert(2);
}
alert(a);

解析：
第一步：预解析（提前找一些东西，找var、function、参数）
从第一行往下面找，找到了var a =1,然后给a初始化一个值a = undefined
再着往下找，找到了function fn1(),然后给fn1初始化一个值，把函数fn1的函数块代码以字符串的形式赋值给fn1，即fn1=function fn1(){alert(2);}
第二步：逐行解读代码
第一行：alert(a) 此时a在预解析的时候赋值为undefined 所以弹出 undefined
第二行：alert(fn1) 弹出了在预解析被赋值的函数块字符串，所以弹出function fn1(){alert(2);}
第三行：给a赋值为1
第四行：运行fn1函数，则弹出2
最后一行：由于第三行a已经被赋值为1，则弹出1

实例二

alert(a);           
var a = 1;           
alert(a);               
function a(){alert(2)}  
alert(a);            
var a =3;                
alert(a);              
function a(){alert(4);} 
alert(a);            
a();

第一步：预解析（提前找一些东西，找var、function、参数）
第二行：初始化 a=undefined
第四行：初始化 a=function a(){alert(2)}
第六行：由于之前a已被函数初始化，此处不能覆盖
第八行：初始化 a=function a(){alert(4);}
第二步：逐行解读代码
第一行：弹出function a(){alert(4);}
第三行：弹出1
第五行：弹出1
第七行：弹出3
第九行：弹出3
第十行：报错（3()不是一个函数）
解析覆盖规则
一旦出现同名的，只留一个，
变量和函数重名只留下函数
函数和函数重名留下后面的函数

实例三

<script>alert(a)</script>
<script>var a =1</script>

此段代码会报错

<script>var a =1</script>
<script>alert(a)</script>

此段代码会弹出1；
script标签作用域规则
script里面定义的变量是全局变量，定义的函数是全局函数。script标签自上向下解析，先处理完一个script 再处理下一个script，所以实例三的第一个例子在第一个script块里面没有初始化a，会报错。在第二个例子中，由于script里面定义的变量是全局变量，所以在第二个script中可以访问到第一个script里面定义的a；

实例四

var a =1;
function fn(a){
   alert(a);   
   a =2;        
}
fn1();
alert(a);   //1

这个例子在第一行a是参数，所以 a初始化为undefined;
所以在第二行弹出undefined;
第一行a的参数也是函数作用域的局部变量，所以运行完fn1后会释放局部变量的空间，所以在最后一行会弹出1。
